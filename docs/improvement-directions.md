# 系统改进方向分析

> 最后更新：2026-06-21
> 本文基于对当前代码的核查（非推测），对照开源个人量化系统（backtrader / vectorbt / vnpy 等）与回测方法论最佳实践，梳理改进方向。

---

## 结论

当前系统在**工程架构**上已属同类个人项目中的上乘水平，短板集中在**回测方法论**与**实盘闭环**两处。

---

## 已验证的现状（基于代码）

- 数据流 `AkShare → DataHub（缓存 + 源 + 刷新任务） → Backtrader Cerebro → 策略注册表（5 个策略） → FastAPI → React`。DataHub 抽象层、策略注册表、18 个测试文件，工程质量扎实。
- 执行采用 `runonce=False`：信号在 `close[0]` 计算、订单于下一根 bar 成交，**不存在未来函数（look-ahead bias）问题**。这是容易踩的坑，当前实现已规避。

---

## 改进方向（按性价比排序）

### 1. 补齐交易成本（最高优先级，改动最小）

全仓库检索不到 `setcommission` / `slippage` —— 回测完全没有手续费、印花税、滑点。

A 股单边印花税 0.05% + 佣金，波段策略 3~10 天换手，成本会显著吃掉收益。**当前所有回测结果均偏乐观。**

> 落地：`cerebro.broker.setcommission(...)`，几行改动，但对结果可信度影响最大。

### 2. 增加风险调整指标与基准对比

`BacktestResult` 当前只输出 `total_return_pct / max_drawdown_pct / win_rate_pct / trade_count`，缺：

- **Sharpe、年化收益、盈亏比**
- **基准对比**：`index_data`（上证指数）已加载但仅用于画图，未用于计算超额收益。

加一个"策略 vs 买入持有基准"的对比几乎零成本，却是判断策略是否真有 alpha 的关键。backtrader 自带 `SharpeRatio` / `Returns` / `SQN` analyzer，直接挂上即可。

### 3. 过拟合防护（方法论硬伤）

当前是"在一段历史上跑一次，看收益率"。业界共识这是不可靠的做法。缺两样：

- **样本外验证 / walk-forward**：数据切成 in-sample 调参 + out-of-sample 验证，最近 10~20% 留作最终 hold-out。
- **参数寻优 + 稳健性检查**：`citic_wave` 有 20+ 个参数（ATR 倍数、各类阈值），无任何 `optstrategy` 寻优，也无参数敏感性分析。参数多而样本外验证缺位，很可能已在历史上过拟合。

### 4. 缓解单股回测的选择偏差

除 `sector_rotation` 外，主路径是单只股票 `adddata`。"在某只股票上跑出高收益"无法说明策略普适性 —— 这是变相的选择偏差。

`backtest/stock_selector.py` 已有批量回测雏形，可扩展为"策略在一篮子股票上的收益分布"，看中位数而非单个样本。

### 5. 实盘闭环（取决于目标定位）

当前系统止步于回测分析台，无信号推送 / 模拟盘 / 券商接口。

- 若定位为**研究工具**，此项可不做。
- 若想真正交易，这是最大缺口。考虑到低频波段（无需盯盘），只需一个**每日收盘后跑信号 + 推送买卖提示**的轻量任务即可，无需引入重型实盘框架（vnpy / nautilus_trader）。

---

## 建议落地顺序

| 顺序 | 项 | 价值 | 投入 |
|------|----|------|------|
| 1 | 交易成本 | 让现有回测结果变可信 | 极小 |
| 2 | 风险指标 + 基准对比 | 判断是否真有 alpha | 小 |
| 3 | 过拟合防护（walk-forward / 寻优） | 让策略选择变可靠 | 中 |
| 4 | 一篮子回测分布 | 消除选择偏差 | 中 |
| 5 | 实盘信号推送 | 打通交易闭环 | 视目标而定 |

第 1、2 项改动小、收益大，应优先；第 3、4 项是中期投入；第 5 项取决于把系统做成研究工具还是实盘系统。
